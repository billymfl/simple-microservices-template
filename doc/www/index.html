Entrypoint for simple-microservice-template, following the 12 factor app design methodology. dotenv is used to load environment variables from .env file into process.env. The .env file should not be committed to a repo as it might contain secrets. The testing library is mocha and chai, with chai-http to test requests. Linting with eslint configured with the google style. config.js contains the environment variables used by the app and the schema to validate the input.
